突破壁垒

发表于 2016-02-18 | 分类于 学习札记 |

如果不能控制这种逃避的倾向，再多、再好的任务管理技巧都是无效的，因为任务中相当重要的一部分（通常因为重要而困难，也因为困难而重要）永远无法完成。所谓效率，是在任务完成之后才能够衡量的。这样看来，对逃避困难>的人来讲，因为没有完整地完成任务，所以无论他们花了多长时间，也都没有效率（相当于分子为零）。

这是李笑来在他的《把时间当朋友》这本书上面第三章中提到的观点，讲的就是：当我们做一件事情的时候，遇到困难的时候，如果直面困难，我们的效率才能说是高的，否则就是一个效率底下的人。其中的观点在书中已经讲的非常明白了。看了他的观点，让我想起来前段时间我在学习 Python 的时候发生的一些事情。

我是在去年的时候开始学习 Python ，一直没有买书本，但是看了《简明 Python 教程》和《笨方法学习 Python 》，还有廖雪峰的《Python2.7 教程》我也看过了一点。看玩这些书本的时候，我对于 Python 的基本语法已经是比较了解了，可以写一些简单的爬虫。但是每当我要写一个比较大的东西，比如 Blog 网站的话，我就会在一些地方被难住了，有的困难经常就是困扰我好几天。面对这些情况，我选择重新学习那些基本的语法，再回头看语法的时候，其实我都能理解，但是再去写的时候就会不知道怎么实现。如此的恶性循环，我甚至觉得我是学不会编程的。

直到现在我看了李笑来的这本书才发现，可能原因并不是我真的不会编程，而是我在这样的一个困难面前没有跨过去，也就是常说的「壁垒」。或者当时的我跨过了这个壁垒，解决了这个困难，那我学习编程的道路就会明朗很多，不会像当初那么糟糕。

有些东西会让我们得心应手，这些不重要，重要的是那些阻碍我们前进的壁垒，只要突破了这些壁垒，我们就能看到新的道路。

这鸡汤，干了，共勉！
