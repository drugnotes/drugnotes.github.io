突破壁垒

发表于 2016-02-18 | 分类于 学习札记 |

如果不能控制这种逃避的倾向，再多、再好的任务管理技巧都是无效的，因为任务中相当重要的一部分（通常因为重要而困难，也因为困难而重要）永远无法完成。所谓效率，是在任务完成之后才能够衡量的。这样看来，对逃避困难>的人来讲，因为没有完整地完成任务，所以无论他们花了多长时间，也都没有效率（相当于分子为零）。

这是李笑来在他的《把时间当朋友》这本书上面第三章中提到的观点，讲的就是：当我们做一件事情的时候，遇到困难的时候，如果直面困难，我们的效率才能说是高的，否则就是一个效率底下的人。其中的观点在书中已经讲的非常明白了。看了他的观点，让我想起来前段时间我在学习 Python 的时候发生的一些事情。

我是在去年的时候开始学习 Python ，一直没有买书本，但是看了《简明 Python 教程》和《笨方法学习 Python 》，还有廖雪峰的《Python2.7 教程》我也看过了一点。看玩这些书本的时候，我对于 Python 的基本语法已经是比较了解了，可以写一些简单的爬虫。但是每当我要写一个比较大的东西，比如 Blog 网站的话，我就会在一些地方被难住了，有的困难经常就是困扰我好几天。面对这些情况，我选择重新学习那些基本的语法，再回头看语法的时候，其实我都能理解，但是再去写的时候就会不知道怎么实现。如此的恶性循环，我甚至觉得我是学不会编程的。

直到现在我看了李笑来的这本书才发现，可能原因并不是我真的不会编程，而是我在这样的一个困难面前没有跨过去，也就是常说的「壁垒」。或者当时的我跨过了这个壁垒，解决了这个困难，那我学习编程的道路就会明朗很多，不会像当初那么糟糕。

有些东西会让我们得心应手，这些不重要，重要的是那些阻碍我们前进的壁垒，只要突破了这些壁垒，我们就能看到新的道路。

这鸡汤，干了，共勉！

正确的转发朋友圈

发表于 2016-02-18 | 分类于 生活随笔 |

我经常会在朋友圈上面转发一些自己觉得有价值的东西，并且可以得避免自己转发一下谣言文章，并且在自己的朋友圈看到谣言性的文章都会提醒好友。

其实要想在朋友圈更加的有价值并不难，首先是从从自己做起，下面是我的几点建议，仅供参考：

文章来源信息

看到一篇文章，我们应该第一时间看一下这篇文章是怎么来的，包括看公众号信息、作者信息；是转载的还是原创？转载的话有没有备注出处？如果这些信息都都是非常模糊不清的话，那么这些文章就不应该转发出去，哪怕这是真实的情况，也会侵犯作者的版权问题。

文章内容

现在谣言穿的最多的就是关于小孩拐卖、养生、支付宝、微信账户信息的，每当看到这些信息的时候就应该提高自己的警惕，看一下这些做大的事情有没有被大型媒体或者身边的亲身经历人经历过，如果没有，那最好也是不要随意转发。

转发信任度较高的信息源

对于那些大型媒体或者一些比较有影响力的大 V 的文章，一般来说都是比较真实，不会放下太大的错误，但是错误还是会有的。虽然一些大型媒体来说也会发一些公关文，但是总比传播谣言要好。

之前我在界面的公众号看过了一篇名为「炸裂！《夏洛特烦恼》居然全片抄袭了《教父》导演的旧作！」的文章，觉得这篇文章还是挺有意思的，所以就转发了这篇文章。没过几天，在别的地方看到说到这篇文章是谣言来的，后来我查了一下，其实不能算是谣言，但是又不能说不是谣言，抄袭的地方还是会有的，只是不能判为全抄袭。

在上面的这个事例中我看到了其实谣言离我们还是挺近的，所以我们更加需要停下来认真的观察清楚这些文章的真实情况。但是我还是会遵守我的那个转发规则：公众号主体明确；标明作者；最重要的还是要符合科学常识。

药品招标如何影响药品的价格

发表于 2016-02-17 | 分类于 移动医疗 |

最近在关注一些关于药品招标和药品价格的问题，其实药品招标只是药品流通的一种形式而已，最初设定这个药品招标政策的目的就是为了患者可以在医院里面买到廉价的药物，但是经过这么多的药品招标，药价并没有实实在在的降下来，那么原因究竟是为什么呢？

首先来了解一下「药品招标」究竟是什么回事。简单来说「药品招标」就是需要采购药品的机构（医院）向符合条件投标人（卖药的，也就是药品代理公司，俗称药贩子）发布当年或者当季要采购的药品名称、数量等信息。投标人就会设定价格和优惠情况给采购机构，采购机构觉得符合自己意愿的采用和该投标人进货。这个过程中投标人可以使用多种方式让采购机构在他那里进货，其中的手段就不一一说明白了。

从中可以简单的看明白了，一个药品要想流通到患者的手里，那么必须要就过三个流程，第一个就是厂家到代理商；第二个就是代理商到医院；第三个就是医院在把药买个患者。先从第一层开始说起，厂家给到代理商的药品加成我们简单的称之为「出厂价」，这个价格基本上就是厂家核算过自己研发、生产、储存、物流等等费用之后得出来的价格。代理商给医院的价格我们称之为「招标价」，「招标价」和「出厂价」肯定是存在差价的，不然代理商吃西北方，至于这个差价又是多少呢，待会再来讨论。最后医院给到患者的药品价格有可能是比「招标价」要高，为什么是有可能呢？因为在过去，医院里面基本都是依靠「药品加成」来给医院创收的，但是最近两三年，有一部分医院已经取消了药品加成。所以那些取消药品加成的医院的药价基本就是医院进货时候的「招标价」了。

其实药厂和最后医院的「药品加成」都不是我们批评药品招标的点，我们要批评的靶子就是「招标价」于「出厂价」之间的差价。曾经多次被媒体报道出来「招标价」比「出厂价」平均高出 5 倍，有的甚至是高出 30-40 倍，这里面的利润都被医药代理公司瓜分已经在招标过程中进行暗箱操作。

这让我想起了之前取消药品限价之后药品升价的新闻，很多普通人在这个时候会把矛头指向药厂，认为这是药厂生产的药品，其中里面的利润肯定是药厂吃了。其实原因并不是这样子的，在调查发现，其中原因有很多，一个是因为原料药涨价，还有就是在过去限价的时候，药厂基本都是硬着头皮去生产这些药品，现在取消限价，涨价也是无可厚非的。其实还有一个原因就是代理公司有囤货现象，但是很多人都看不多这一面，只是知道药厂一定就是奸商。就好比春运火车票被黄牛党囤起来了，就把矛头指向中国铁路局。

关于国内药品流通的问题，还是有很多东西值得我们去认真思考的，或者我的这些思考都是错的。

文章排版的重要性

发表于 2016-02-16 | 分类于 学习札记 |

作为一个 Blogger ，而且是坚持原创、对阅读有追求的 Blogger ，在看一篇文章的过程中我经常会留意整篇文章的排版，并且在自己写作的过程中避免犯下低级的排版问题。虽然我现在的排版并不是特别的标准，但总算是往好的方向发展，这一点让我感到很满意。

那么一篇文章为什么要那么注意排版呢？写出来用户能看明白不就可以了吗？其实不是的，排版混乱的文章不仅让用户理解起来费劲，而且很容易分散读者的注意力，不利于深度阅读。就算是用户能够理解文章的内容，但是排版依旧重要。

第一，文章就是产品

其实每一篇文章就是一个产品，而作者就是一个产品经理，换句话说就是「文章就是你的孩子」。作为一个产品经理，你应该对自己的产品花点心思，尽可能的追求完美，因为只有这样的产品经理才能打造出简单易用、交互有好的世界级产品。所以作为一个写作者，对自己的每一篇文章也应该像产品经理一样对待自己的产品，力求自己的文章不仅内容好，排版也好，让读者阅读起来无阻碍。

在我的产品观里面，简洁就是最好的没，没有设计就是最好的设计。所以我会很少在我的文章上面插入一些表情包、逗逼图，我觉得这是一个低俗的行为。其实在微信、QQ 聊天上面，我也是非常抵制这些表情图片的。我觉得都在发这样的表情图，严重的阻碍了聊天的进行，这是对对方的不尊重，浪费时间的行为。

第二，好的排版更利于深度阅读

我觉得好的内容就应该得到尊重，就应该值得花时间进行深度的阅读。这些好的文章给人的唯一作用就是「思考」，或者说「Thinking」。所以这些好内容呈现的时候就应该考虑到让用户不被外部因素干扰，而是专注在文字上面。文章可以出现一些图片内容，用来缓解用户的疲劳或者给用户一些思考。但是不应该大篇幅的出现逗逼图，我再说一次：「不要出现逗逼图」。当然，如果你写的是类似于「嗅事百科」这种类文章，这种类型的文章可以用大量的逗逼图，但是这类文章不是我们今天讨论的内容。

第三，排版也是写作者应该修炼的技能

很多人之所以写作，其实更的是为了锻炼自己的思维能力。一篇文章的内容很重要，内容的呈现方式也很重要。这就跟我们读书时候用笔写作文一样，如果的的文章内容写的很好，但是你的字体出奇的丑，那么你的文章肯定得不到高分。互联网时代，我们已经很少提笔写字了，所以排版的设计就体现了一个人对于文字展示的思考，排版可以锻炼人一种文字展示的技能，驱使写作者去思考怎么样的排版才会让用户体验更好，就跟怎么样的字能得到阅卷老师给的高分。

那么怎么样的排版才是好的排版？

上面提到了几点关于排版重要性的问题，下面就应该来点干货的，说一下我对于排版的建议：

  1. 如果你使用的是 WordPress 博客，那么我建议你把你的编辑器修改为支持 MarkDown 语法，支持这个语法之后你就可以安心的写作，把排版的问题交给 MarkDown 就好了。说实在的， WordPress 自带的编辑器写出来的文章排版真的是没法看，不仅不方便，写出来还难看。那么怎么才能让 WordPress 支持 MarkDown 语法呢？这个当然就是自行 Google 啦，可以选择使用插件，也可以通过修改代码，本人推荐使用插件来实现，我只能帮你到这里了。当你的 WordPress 网站支持 MarkDown 语法之后，你应该话半个小时去学习一个 MarkDown 语法，链接直达 Markdown 语法说明 (简体中文版)

  2. 除此之外，我还建议你看一下由知乎网友 Hindy 在他的知乎周刊上面发表的文章写给大家看的中文排版指南，上面讲述了平时写作的时候会遇到的一些排版问题，非常实用。

浅谈互联网分享

发表于 2016-02-12 | 分类于 互联网 |

2007 年年初，我开始接触互联网，那时候 QQ 正在兴起，而我还是一个上初一的学生，当时用 QQ 加了很多身边的同学朋友，无外乎就是看看 QQ 空间，联系一个身边的朋友。

那时候的我对于互联网的本质没有太大的认识。

后来我开始在互联网上面接触一些除了 QQ 之外的东西，甚至开始接触移动互联网。在 2008 年的时候，我开始拥有了第一台手机，而且是带有上网功能的，当时浏览最多的手机网站就是 3G 门户和乐讯网。当时浏览 3G 门户纯粹就是为了玩手机版的 QQ ，而乐讯网就是为了看一下关于手机的东西。乐讯网上面当时就是一个依靠用户生产内容的移动门户网站，奇怪的是当时我并不了解网易、新浪这些老牌的门户网站，最大的原因应该是家里一直没有一台电脑。

那个时候的我还是不能理解互联网在用户心目中的本质。

2011 年，当时我在读高一，开始接触微博，并且在早些时候就已经开通了新浪博客，但是上面很少写东西，一直就是这样荒废了。当时觉得新浪微博会是一个互联网的一个新的浪潮，限于当时自己并没有生产内容的能力和时间，所以这次中国互联网浪潮错过了，后来在微信公众平台推出来的时候，我觉得这会是一个比新浪微博更大的浪潮，当时我还在都大一。

2013 年 7 月，我开通了一个微信公众号，一个订阅号来的，先要做一个校园内部的独立商城，以微信为入口，后来在申请自定义菜单的时候，发现微信团队没有给没有认证的订阅号开放自定义才能，但是服务号可以有，所以就在申请了一个服务号。

但是我也在接触开源系统 ECshop ，因为要搭建校园内部的独立商城系统，并且将这个商城系统对接到微信公众上面。这样我就会牵涉到微信公众号的开发模式，对于完全没有接触过代码的我来说，没有任何的头绪，只能到网络上面去找相关的教程。

走了很多的弯道，我找到了一个名字叫做 「李捷」的人在 YY 做了三节课分享微信开发教程，当时讲到的内容比较偏向技术，没有很好的结合业务，但是让我对微信公众号开发有了一定的了解，比如我知道可以用 PHP 来写。

后面的时间我再去找相关的教程，还真的被我找到了，找到一个叫做 「易伟」，据说是一个律师，他做了一系列关于微信公众号相关功能的讲解，基本就是一个功能一段代码去实现。当时他的视频给像我这种菜鸟起到了很大的作用，后来他还出了一本书，好像书名叫做《微信公众号开发接口实践》什么的，忘记了，书本首发的当天我就把书买了回来。后来 「青龙老贼」在他的公众号上面也做了一套类似 「易伟」这样教程，只是牵涉到的功能上面有点不一样而已，还有就是 「易伟」实在百度 BCE 上面部署代码的， 「青龙老贼」是在新浪 SAE 上面部署代码。

通过学习微信开发，我学会了下面这些东西：
1. 知道了 PHP 代码除了做网站，还可以做别的事情，比如微信公众号开发。
2. 原来有 BCA 和 SAE 这种低成本的服务器。（PS：好像不应该说是服务器，但是当时我就是这么理解的）
3. 前面的都不重要，重要的是原来分享之后可以或者一些很意外的收获，比如 「易伟」出了本书， 「青龙老贼」加了不少分析， 「李捷」为他所在工作的培训机构 「LAMP 兄弟连」获得不少的知名度。

于是我就搭建了我自己的 WordPress 博客，并且尝试着去写一些对自己有价值的东西，因为这些对自己有价值的东西也会对某些人有价值。

生活的选择

发表于 2016-02-09 | 分类于 生活随笔 |

今天下午，约了几个同学一起去探望初三时候的班主任，谈了一个下午，谈到最多并且让我印象最为深刻的就是「选择怎么样的生活？」这个话题上。

我的班主任是1980年出生的，大学毕业之后就一直留在我们镇上的高中教书，教过我们生物，也教过我们化学，但是他说他中学时候是语文最好的。

他向我们讲述他在刚刚毕业的时候那种心酸的日子，在学校教书的工资是非常低的，也谈不上什么福利。最后他还说「其实那时候的老师都是熬过去的」，听到这句话的时候我是非常震撼的，因为我无法理解他们为什么愿意这样子去熬，教师给到他们的前景又是如何？

现在从他口中得知，他对于他现在的生活是非常满意的，因为最近几年工资都升了，福利也变好了，有儿女，并且还在县城里面供有一套房子，钱不是很多，但是日子确实可以过的很滋润。这个时候我终于理解了他当初选择「熬下去」的理由了，但是我还是不会像他那样选择，选择过这样的日子。或者仅仅是因为我还很年轻，刚刚步入社会，对于真实的生活没有很深切的理解。

在他的口中知道他很满意现在的生活，跟我所理解的生活有了很大的出入。因为在我的内心里面，永远都不会满足于每天重复着一样的日子，也接受不了「平平淡淡就是福」这句话。或者「平平淡淡就是福」这样的生活才是大多数人选择的生活，也是相对来说比较正确的选择，但对于我来说并不是最好的选择，我只求最好、最合适自己的生活，而不是绝大多人认为是正确的生活。

年味(1)

发表于 2016-02-08 | 分类于 生活随笔 |

今天是大年初一，前几天回家之后一直都在忙着过年的事情，走访亲戚、准备年货、打扫卫生等等工作，忙的不亦乐乎。今天终于有时间坐下来好的的读书写字了，所以更新一下博文，来谈谈我自己所经历的过年的味道。

现在我已经二十多岁了，不能说完全的是一个支撑一个家庭的大人，但也算是一个大人。现在的我和几年前的我对于过年的意义肯定是不一样的，自己一直都在留意着这几年家里过年的一些变化的地方，现在希望把这些变化凭借我自己的记忆整理起来，写在这里。

今天就先来谈谈关于过年逛集市的变化

「逛集市」应该是别的地方的说法，还有的地方称之为「赶集」，「赶集」这个词语应该是我在小学时候的语文课本上面见到过的，现在已经忘记了是哪个作者写的文章了。「逛集市」我的老家叫做「趁墟」，好像白话也是这么说的，其实大概的意思就是「去热闹的地方凑热闹」。

因为家住农村，而且是一个相对比较落后的贫困县的一个小镇上面，所以在我很小的时候「趁墟」的机会是非常的少的，因为「趁墟」就必须要买点什么东西吧，那就要花钱。所以家里好一点条件的人的话就可以经常去，条件不好的人的话就只能一个月去一两次，甚至一年一次，那就是年底的时候去一次。所以其实「趁墟」对于小时候的我们也是一种非常不错的活动。

县城里面有好几个镇，其中就用农历来计算，把尾数为147的天数归为其中三个镇的「趁墟日」，把尾数为258的天数归为另外三个镇的「趁墟日」；再把尾数为369的天数归为其中镇的「趁墟日」；我所在的镇就是把258为「趁墟日」的，所以一个月就有了9个「趁墟日」。但是在那个时候，一年能去几趟「趁墟」已经是很不错的。

其实「趁墟」上面不仅是卖东西的，对于我们学生来说最吸引人的地方就是「趁墟」上面有游戏机室，那时候是五毛钱两个硬币，一个硬币就可以玩一局，我们玩的最多的就是「恐龙快打」、「三国志」、「拳王」、这些游戏，因为都是属于闯关游戏，所以打得好的人就可以一个硬币打爆机，那时候常常把家里人给的零用钱用来打机。记得有一个下午，我们的一个同学拿着五块钱去打，打了一个下午，天快要黑的时候才打完，那时候发现自己点击键盘的手已经是弯曲变形的，现在想起来都觉得好笑。

无论是哪个镇，在年底的时候都会有一个最后的「趁墟日」，比如我们镇就是年廿八，这是一年中最后一个「趁墟日」了，镇上的很多人都会选择在这一天置办年货、给孩子买衣服等等。那一天就是镇上「趁墟」里面最旺的一天，往往就是水泄不通的，类似于现在的双十一阿里服务器崩溃一样。

在我读都初中之前，镇上的人都是非常看重这个「趁墟日」的，所以那天在马路上经常就是成群结伴的人走在一起或者是坐在拖拉机的货箱上面，因为那是后没有多少人是有摩托车的，跟不要说是小车了。就这样一群人被拉到了「趁墟」，大家开始买年货，买衣服，买玩具，买各种各样的东西，到了下午太阳快要下山的时候，大家就会不约而同的出现在拖拉机停车的地方，准备回家。

很多年过去了，都虽然「趁墟日」保留了下来，但是那种全村人去「趁墟」的场景已经很少能看到了，因为大家都有了小车、摩托车，到「趁墟」上面买东西时非常方便的，想什么时候出去就什么时候出去，非常的方便。

现在过年没有了童年时期那种热闹，很多东西会被时间慢慢的消磨，至今为止，被消磨掉的不仅仅是「趁墟」这个东西，还有很多，以至于我记不起来都有哪些。

如何让技术主动亲吻运营

发表于 2016-02-05 | 分类于 互联网 |

这篇文章首发在我的微信公共号「空谷幽兰」，微信号为 DrugNotes ，大家可以关注一下

前段时间在微信公众号上面看了一篇文章，文章的名字叫做「技术人员愿意亲吻怎么样的运营」。文章通过分析「运营人员为什么要和技术人员建立好关系？」以及「运营人员如何跟技术人员建立好关系？」，还有文章最后简单的提到了「运营人员眼中好的研发是怎么样的？」来分析了运营人员如何和技术搞好关系，推进产品需求的完善。

在我的记忆中，有的公司的产品、技术、项目以及运营是在一个区域办公的，可能很多互联网公司就是这几个部门组合起来运转的。产品和技术在一起办公室非常正常的，那么运营和技术有没有必要在一起办公呢？我觉得这个非常有必要，以为发现、提出需求以及发现 bug 的很多都是运营人员，如果在一起办公的话那就大大的缩减了大家沟通的距离，减少的部门之间的沟通障碍。

话谈回来，运营人员应该怎么做才能让技术人员“亲吻”自己呢？让自己的需求合理的被产品经理和技术接受，并且投入开发，保证业务的开展。

你的需求是合理的

需求合理是一个运营人员最基本的保证，并且保持你的需求是一贯合理的，长期以来才能得到产品和技术的信任，以确保和项目、产品、技术的良好合作关系。如果运营人员提出一个需求，产品和技术熬夜开发、测试、上线之后，发现业绩没有上去就算的，你他呀用都没有用过。这时候的产品和技术就会觉得自己当初被当作是傻逼了，心里面很是难受，就会觉得这个运营同事是如此的不靠谱。所以对于一个运营人员，在提出一个需求的时候要确保这个需求对于业务来说是非常有必要，而且回报是可以预测的，并且考虑到功能上线后可能产生的一些不好的事情，需要权衡整个流程的利弊，避免开发了一堆没用的东西。

全程参与需求开发

虽然运营属于业务方，但是也需要及时的跟进产品和技术的开发工作，避免产品和技术因为理解错误而做出于需求无关的事情。一个新功能上线之前，一般都会在运营人员加入测试，但是最好不要再功能上线的那天才去了解这个功能的进展，越早了解就会越好，避免在上线之前出现意外，技术又要通宵修复代码。其实上线前的测试很多时候产品经理已经测过好几遍了，但是运营人员还是需要参与进来，并且认真的对待测试，因为整个公司里面，没有那个部门会对产品的流程比运营更加熟悉了，那些非常细微的错误往往只有运营人员才之后。虽然产品是产品经理的孩子，但是只有运营人员才能深度理解产品的业务流程。在参与这个测试过程中，还可以跟技术和产品打成一片，因为一次上线就会需要牵涉很多的人员，大家就会在这个过程中熟络。

在站在技术的角度思考问题

这里要谈的不是要我们运营人员去学点技术，而是要懂得理解技术人员的内心变化。有些运营人员发现了 bug 的时候就感觉自己发现了新大陆一样，内心无比的开心，而且乐呵呵的跑过去跟技术反馈：「我擦，这么简单代码都有 bug，还好我发现了」，然后得以的回到工作岗位去。其实技术人员最不愿意看到自己的代码里面有 bug，尤其是被不擅长编程的运营傻逼发现了。当你告诉他这里有 bug 的时候他是非常不愿意接受这个事实的，那应该怎么办呢？其实最好的就是先把这个事情和产品经理沟通，如果实在紧迫的话，跟技术人员沟通的时候直接说有问题就可以了，就算知道是什么原因导致的，也不要说出来，其实技术看到问题就会知道怎么排查原因的，而且千万不能面带笑容。

善用技术的力量

在「技术人员愿意亲吻怎么样的运营」这边文章中也提到了运营人员最好能学点技术，比如PHP、CSS、JS等基本知识。我觉得懂点技术还不够，要善用技术的力量。就如上面提到了，不能再跟技术人员沟通的时候秀自己，就算知道了这个问题的解决思路或者这个 bug 技术层面的问题，都不要一开始就跟技术说出来，除非你的编程能力在被技术的同时认可（一些就是开发转运营的的技术大牛）。那要技术来干嘛？其实的作用就是在自己的本职工作中起到辅佐的作用，比如评估一个需求技术能不能满足？需求是否合理等问题。避免因为自己毫无技术概念的情况下提出一些让产品经理和开发人员冷笑的需求。

好的运营会从技术人员最大化的利益点去出发，提出需求，推进开发，保证业务的正常开展。无论是通过什么方式，确保需求能够满足，并且和产品、技术同时的良好合作关系，那你就是一个好运营。

一个好的运营人员，是一个能推动技术完善需求的产品经理。
