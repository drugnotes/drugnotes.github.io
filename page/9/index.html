生活的选择

发表于 2016-02-09 | 分类于 生活随笔 |

今天下午，约了几个同学一起去探望初三时候的班主任，谈了一个下午，谈到最多并且让我印象最为深刻的就是「选择怎么样的生活？」这个话题上。

我的班主任是1980年出生的，大学毕业之后就一直留在我们镇上的高中教书，教过我们生物，也教过我们化学，但是他说他中学时候是语文最好的。

他向我们讲述他在刚刚毕业的时候那种心酸的日子，在学校教书的工资是非常低的，也谈不上什么福利。最后他还说「其实那时候的老师都是熬过去的」，听到这句话的时候我是非常震撼的，因为我无法理解他们为什么愿意这样子去熬，教师给到他们的前景又是如何？

现在从他口中得知，他对于他现在的生活是非常满意的，因为最近几年工资都升了，福利也变好了，有儿女，并且还在县城里面供有一套房子，钱不是很多，但是日子确实可以过的很滋润。这个时候我终于理解了他当初选择「熬下去」的理由了，但是我还是不会像他那样选择，选择过这样的日子。或者仅仅是因为我还很年轻，刚刚步入社会，对于真实的生活没有很深切的理解。

在他的口中知道他很满意现在的生活，跟我所理解的生活有了很大的出入。因为在我的内心里面，永远都不会满足于每天重复着一样的日子，也接受不了「平平淡淡就是福」这句话。或者「平平淡淡就是福」这样的生活才是大多数人选择的生活，也是相对来说比较正确的选择，但对于我来说并不是最好的选择，我只求最好、最合适自己的生活，而不是绝大多人认为是正确的生活。

年味(1)

发表于 2016-02-08 | 分类于 生活随笔 |

今天是大年初一，前几天回家之后一直都在忙着过年的事情，走访亲戚、准备年货、打扫卫生等等工作，忙的不亦乐乎。今天终于有时间坐下来好的的读书写字了，所以更新一下博文，来谈谈我自己所经历的过年的味道。

现在我已经二十多岁了，不能说完全的是一个支撑一个家庭的大人，但也算是一个大人。现在的我和几年前的我对于过年的意义肯定是不一样的，自己一直都在留意着这几年家里过年的一些变化的地方，现在希望把这些变化凭借我自己的记忆整理起来，写在这里。

今天就先来谈谈关于过年逛集市的变化

「逛集市」应该是别的地方的说法，还有的地方称之为「赶集」，「赶集」这个词语应该是我在小学时候的语文课本上面见到过的，现在已经忘记了是哪个作者写的文章了。「逛集市」我的老家叫做「趁墟」，好像白话也是这么说的，其实大概的意思就是「去热闹的地方凑热闹」。

因为家住农村，而且是一个相对比较落后的贫困县的一个小镇上面，所以在我很小的时候「趁墟」的机会是非常的少的，因为「趁墟」就必须要买点什么东西吧，那就要花钱。所以家里好一点条件的人的话就可以经常去，条件不好的人的话就只能一个月去一两次，甚至一年一次，那就是年底的时候去一次。所以其实「趁墟」对于小时候的我们也是一种非常不错的活动。

县城里面有好几个镇，其中就用农历来计算，把尾数为147的天数归为其中三个镇的「趁墟日」，把尾数为258的天数归为另外三个镇的「趁墟日」；再把尾数为369的天数归为其中镇的「趁墟日」；我所在的镇就是把258为「趁墟日」的，所以一个月就有了9个「趁墟日」。但是在那个时候，一年能去几趟「趁墟」已经是很不错的。

其实「趁墟」上面不仅是卖东西的，对于我们学生来说最吸引人的地方就是「趁墟」上面有游戏机室，那时候是五毛钱两个硬币，一个硬币就可以玩一局，我们玩的最多的就是「恐龙快打」、「三国志」、「拳王」、这些游戏，因为都是属于闯关游戏，所以打得好的人就可以一个硬币打爆机，那时候常常把家里人给的零用钱用来打机。记得有一个下午，我们的一个同学拿着五块钱去打，打了一个下午，天快要黑的时候才打完，那时候发现自己点击键盘的手已经是弯曲变形的，现在想起来都觉得好笑。

无论是哪个镇，在年底的时候都会有一个最后的「趁墟日」，比如我们镇就是年廿八，这是一年中最后一个「趁墟日」了，镇上的很多人都会选择在这一天置办年货、给孩子买衣服等等。那一天就是镇上「趁墟」里面最旺的一天，往往就是水泄不通的，类似于现在的双十一阿里服务器崩溃一样。

在我读都初中之前，镇上的人都是非常看重这个「趁墟日」的，所以那天在马路上经常就是成群结伴的人走在一起或者是坐在拖拉机的货箱上面，因为那是后没有多少人是有摩托车的，跟不要说是小车了。就这样一群人被拉到了「趁墟」，大家开始买年货，买衣服，买玩具，买各种各样的东西，到了下午太阳快要下山的时候，大家就会不约而同的出现在拖拉机停车的地方，准备回家。

很多年过去了，都虽然「趁墟日」保留了下来，但是那种全村人去「趁墟」的场景已经很少能看到了，因为大家都有了小车、摩托车，到「趁墟」上面买东西时非常方便的，想什么时候出去就什么时候出去，非常的方便。

现在过年没有了童年时期那种热闹，很多东西会被时间慢慢的消磨，至今为止，被消磨掉的不仅仅是「趁墟」这个东西，还有很多，以至于我记不起来都有哪些。

如何让技术主动亲吻运营

发表于 2016-02-05 | 分类于 互联网 |

这篇文章首发在我的微信公共号「空谷幽兰」，微信号为 DrugNotes ，大家可以关注一下

前段时间在微信公众号上面看了一篇文章，文章的名字叫做「技术人员愿意亲吻怎么样的运营」。文章通过分析「运营人员为什么要和技术人员建立好关系？」以及「运营人员如何跟技术人员建立好关系？」，还有文章最后简单的提到了「运营人员眼中好的研发是怎么样的？」来分析了运营人员如何和技术搞好关系，推进产品需求的完善。

在我的记忆中，有的公司的产品、技术、项目以及运营是在一个区域办公的，可能很多互联网公司就是这几个部门组合起来运转的。产品和技术在一起办公室非常正常的，那么运营和技术有没有必要在一起办公呢？我觉得这个非常有必要，以为发现、提出需求以及发现 bug 的很多都是运营人员，如果在一起办公的话那就大大的缩减了大家沟通的距离，减少的部门之间的沟通障碍。

话谈回来，运营人员应该怎么做才能让技术人员“亲吻”自己呢？让自己的需求合理的被产品经理和技术接受，并且投入开发，保证业务的开展。

你的需求是合理的

需求合理是一个运营人员最基本的保证，并且保持你的需求是一贯合理的，长期以来才能得到产品和技术的信任，以确保和项目、产品、技术的良好合作关系。如果运营人员提出一个需求，产品和技术熬夜开发、测试、上线之后，发现业绩没有上去就算的，你他呀用都没有用过。这时候的产品和技术就会觉得自己当初被当作是傻逼了，心里面很是难受，就会觉得这个运营同事是如此的不靠谱。所以对于一个运营人员，在提出一个需求的时候要确保这个需求对于业务来说是非常有必要，而且回报是可以预测的，并且考虑到功能上线后可能产生的一些不好的事情，需要权衡整个流程的利弊，避免开发了一堆没用的东西。

全程参与需求开发

虽然运营属于业务方，但是也需要及时的跟进产品和技术的开发工作，避免产品和技术因为理解错误而做出于需求无关的事情。一个新功能上线之前，一般都会在运营人员加入测试，但是最好不要再功能上线的那天才去了解这个功能的进展，越早了解就会越好，避免在上线之前出现意外，技术又要通宵修复代码。其实上线前的测试很多时候产品经理已经测过好几遍了，但是运营人员还是需要参与进来，并且认真的对待测试，因为整个公司里面，没有那个部门会对产品的流程比运营更加熟悉了，那些非常细微的错误往往只有运营人员才之后。虽然产品是产品经理的孩子，但是只有运营人员才能深度理解产品的业务流程。在参与这个测试过程中，还可以跟技术和产品打成一片，因为一次上线就会需要牵涉很多的人员，大家就会在这个过程中熟络。

在站在技术的角度思考问题

这里要谈的不是要我们运营人员去学点技术，而是要懂得理解技术人员的内心变化。有些运营人员发现了 bug 的时候就感觉自己发现了新大陆一样，内心无比的开心，而且乐呵呵的跑过去跟技术反馈：「我擦，这么简单代码都有 bug，还好我发现了」，然后得以的回到工作岗位去。其实技术人员最不愿意看到自己的代码里面有 bug，尤其是被不擅长编程的运营傻逼发现了。当你告诉他这里有 bug 的时候他是非常不愿意接受这个事实的，那应该怎么办呢？其实最好的就是先把这个事情和产品经理沟通，如果实在紧迫的话，跟技术人员沟通的时候直接说有问题就可以了，就算知道是什么原因导致的，也不要说出来，其实技术看到问题就会知道怎么排查原因的，而且千万不能面带笑容。

善用技术的力量

在「技术人员愿意亲吻怎么样的运营」这边文章中也提到了运营人员最好能学点技术，比如PHP、CSS、JS等基本知识。我觉得懂点技术还不够，要善用技术的力量。就如上面提到了，不能再跟技术人员沟通的时候秀自己，就算知道了这个问题的解决思路或者这个 bug 技术层面的问题，都不要一开始就跟技术说出来，除非你的编程能力在被技术的同时认可（一些就是开发转运营的的技术大牛）。那要技术来干嘛？其实的作用就是在自己的本职工作中起到辅佐的作用，比如评估一个需求技术能不能满足？需求是否合理等问题。避免因为自己毫无技术概念的情况下提出一些让产品经理和开发人员冷笑的需求。

好的运营会从技术人员最大化的利益点去出发，提出需求，推进开发，保证业务的正常开展。无论是通过什么方式，确保需求能够满足，并且和产品、技术同时的良好合作关系，那你就是一个好运营。

一个好的运营人员，是一个能推动技术完善需求的产品经理。

防止产品抄袭

发表于 2016-02-05 | 分类于 互联网 |

在现在的互联网行业，创业门槛已经变得越来越低，在执行力强的团队里面，一个 Idea 的产生到产品上线估计也就一头半个月的时间，快速一点的就是几天时间。对于很多大公司来说，比如 BAT ，响应速度更加是惊人。而且在这个开源时代，技术已经不是什么可以隐藏得了的东西，只要是在 Google 、维基百科以及书本就能学到的东西。

所以就会很多准备创业的朋友担心自己“想出来”的 Idea 会不会在产品刚出来的时候会发现这块大市场，然后大公司就会派一个团队去模仿也做一个，这样是不是就是死翘翘了，我觉得有这样想法的人还是不要创业算了。虽然这话说的有点过，但你可以参考一下我下面的这些分析。

1. 大公司更加愿意收购而不是模仿

对于一个新的产品、新的市场，就是前期被大公司看上了，大公司也不一定会采用模仿的形式去开拓自己在这一个领域的市场，更多的是通过投资的方式将产业进行整合。比如说当时的滴滴和快滴起步的时候，市场前景也是很不错，但是腾讯和阿里这样的大公司并不会说自己模仿着这两家公司然后做一个同类型的产品，而是选择投资这两家公司。并且通过自身业务的整合，加快这个新行业的发展的同时还帮助了原有公司的业务（微信和支付宝红包就是在滴滴和快滴的补贴战中推广开来的）。其实收购的形式是非常合理的，因为本来人家已经起步了，如果加进去，就难免会有竞争，还不如投资一下，从中得到好处。

2. 忽略产品的开发

之前我有个朋友想要做一个快递类的工具，他觉得他的 Idea 是非常好的，所以从一开始就担心自己的商业模式回被大公司抄袭，所以整天在研究怎么对应抄袭，但是忽略了产品本身，最后几个月过去了，产品还是没有成功开发出来。其实作为一个创业者，更加应该关注自己的产品本身，而不是外部的竞争因素。说白了就是在这个市场上面，这个初创小公司根本就不在那是大公司的眼里。

所以，别想着你的 Idea 有多好，应该想着怎么把你的产品做好。

Never Too Late ?

发表于 2016-02-03 | 分类于 生活随笔 |

由于本人一直都在关注自己日常的工作，并且会使用 GTD 去管理自己的时间，并且通过一些软件或者是 Chrome 浏览器上面的插件来给自己的时间管理作为工具，减少玩手机的时间，提高工作效率。

无论是过去还是现在，我都还是很少在微信上面和朋友们闲聊，第一个原因是我没有很多的朋友；第二个原因是因为我很少花时间在微信上面聊天，所以我的朋友越来越少。我现在还是坚信这样的一句话「社交不会带来有用的东西，社交当中获取的信息才是有用的」。所以我现在还是很少在微信里面聊天，但是朋友圈却是刷的比较频繁，其实这就是我今天要讲的东西。

在工作日的时候，我会把自己手头上的时间安排的满满的，甚至是在下班回家之后还是加班，熬夜学习，这些事情在工作日的时候是一成不变，雷打不动的。只是那些永远也学不完的东西在下班回家之后零星时间显得有点不足，所以就会希望自己能在周末的时候认认真真的学习，似乎会有很多的要学习的东西排在我的学习日程里面。

但是每到了周末，我就会选择把自己放轻松，然后就是大块时间用来刷朋友圈，剩下来学习的时间并不比工作日的时候多多少，然后步入新的一周的时候，发现上个星期的学习任务并没有完成，显得自己的学习进程其实变慢了。

其实这样的例子就如同写作一样，总是会觉得工作日的时候没有太多的时间用来写作，等到有空、等到周末的时候就是大笔一挥，打开 Blog 后台，打开微信公众号平台，然后就拿起手机耍朋友圈。

所以，其实做任何事情都不应等到有时间了再去做，更何况是学习，写作这样长期战斗的工作，更不能说要空出大片的时间来做，并且之外一次做好。

今天在阅读潇洒姐的 《女人明白要趁早》这本书，书里面提到的是潇洒姐在前几年做的一个每天专注 3 小时的事情，有他妈的给我打了鸡血，所以今天才写了这篇文章。

其实，潇洒姐在她的书里面还说到了 「Never too late 」这句话其实是错的，好吧，我上面写的也是错的。

XAMPP 启动 Apache 失败的解决方案

发表于 2016-01-23 | 分类于 学习札记 |

最近在学习 PHP，在使用 XAMPP 软件搭建 PHP环境之后，尝试启动 Apache 的时候却发现失败了，返回的信息如下：

一时间自己也不知道是怎么回事，第一时间想到的就是端口的问题，后来上 Google 查了一些资料，可能就真的是80端口的问题，后来就适当的调整了一下端口的配置。现在把这个过程记录下来，希望对自己以后的学习和初学者们能有帮助。

第1步
从 XAMPP 的控制面板，选择Apache的配置，就是 Start 后面的 config 选项，并选择 httpd.conf 文件。然后再 httpd.conf 文件里面找到 listen 80，把 80 改为 8080 这类数值，然后把文件保存。

第2步
同样的操作，在 Apache 下，再次点击配置按钮，但这次选择了 httpd-ssl.conf 这个文件，在 httpd-ssl.conf 文件中找到，发现一行写着 listen 443，并且叫 443 改为别的数字，比如 4433，然后保存文件。

并改变 443 到任何你想要的端口。我将使用 4433作为新的端口号。

然后再重新启动 Apache 就可以成功了。

零基础学习SQL的资源推荐

发表于 2016-01-05 | 分类于 学习札记 |

最近在学习SQL，一来可以帮助自己快速的处理日常工作，而来为了以后自己做数据分析做点基础的工作。在没有人教我的情况下，我上网找了一些资料进行自主学习，其中发现了一些比较好的学习资源可以分享出来。主要就是分享两个网站和一本非常基础的书。

前提：我是在Access上面进行实践，因为原本电脑就有这个软件，而且操作简单友好，SQL初学者也可以像我一样考虑使用Access。

推荐的第一个网站就是W3CSQL基础教程上面的SQL教程，相信很多人的第一门自学的编程语言都是在上面学习的，上面的SQL教程也是非常的简单易学，用一个小时就可以SQL基本的概念和语法理解了，后面还有20道选择题来检查一下自己的收获，值得推荐。

另外一个网站也是类似于W3school的，不过貌似更加的深入一些，相对来说比W3school会更加多内容一点，但是讲解没有W3school那么详细。那是在1keydata上面的SQL教程，看完W3school的教程之后，可以回去看这上面的内容，相当于复习。也可以在后面的过程中将这两个网站的教程做个对比，加深对于SQL的理解。

最后就是推荐一本书，书的名字叫做《SQL必知必会》。这本书也是关于SQL一些最基本的概念，语法等等内容，对于初学者有非常大的帮助。这本书的特点除了基础、全面之外，最大的好处就是会在讲解的过程会对不同的数据库不同的操作列举出来，大大的减少了初学者走弯路。同时这本书还可以在后面加强SQL技能的时候作用查阅书籍来用，遇到不懂得地方可以按照目录来查找解决的方案。

以上就是我自己初学SQL的一些收获，或者以后会有更新。

2015 年最后的夜晚

发表于 2015-12-31 | 分类于 生活随笔 |

今天是2015年12月31日，就在不久的刚才，从公司加班回来的时候已经是晚上23点多。这一年，过的紧凑而且有规律，虽然难免会有磕磕碰碰，一时发热的。但总算是按照计划在前进，虽然没有意料中的快。

2015年，没有很宏大的目标，我只是想找到一份自己想要的工作，先不考虑公司和薪资，只要能让自己进入医药电商行业就可以的。如愿以偿4月份我投递出了我的第一份实习简历，最后顺利的拿到了七乐康offer。从入职到现在，有的满腔热血，也有过心灰意冷；有过怀疑，但却满怀希望；有个欢聚，也有过离别；有过想要离开，但最后还是留了下来。但是心中的那股热情也就还在，未曾减退。渐渐的明白了其实那只是每个职场新人必须要面对的那个阶段，这个阶段就是所谓的NObody，而我自己需要去改别，去做出努力的就是怎么快速的从这个NObody的现状中突破出来，成为Somebody。这也是我一直努力的方向。

2015年，虽然没有按照的计划的来每个月完成相应的阅读，但却一直保持着阅读。在这一年里面，阅读成为了我日常的一件离不开的事，或者那是看过的知识最后都已经烟消云散，但是留在记忆深处的可能是阅读过程中的顿悟。

2015年，我开始用日记来记录着我自己的生活，那些流水账式的写作，没有给我留下太多的惊喜，也没有为我的写作能力提升多少。从来不会去看自己每天写下来的日记，也不记得里面具体记录了那些内容。或者会在那么一天回去看得时候，发现里面记录下来的事情竟然是如此的陌生，似乎没有发生过一样。

2015年尝试着将自己从广泛撒网当中转到重点捕鱼上面，但是在这个过程中发现自己撒的网还不过广，需要汲取更多的额外知识，才能去提炼自己最为关键的竞争力。

2015年最失败的事情就是没有真正的将自己学习过的Python的只是做出来一个有用的东西，理论知识学了很多，但是却没能持续有效的去投入做一个有用的东西。这样会导致自己后面的数据分析受到一定的阻碍。

最后预告，明天完成2016年的新愿望。
