互联网并没有改变信息不对称

发表于 2017-09-02 | 分类于 互联网 |

李文星之死，也揭开 Boss 直聘在履行平台审核上的漏洞，媒体和同行纷纷将舆论的矛头指向 Boss 直聘，对 Boss 直聘进行口诛笔伐。

Boss 直聘没有履行好平台的责任，确实存在责任，应该得到惩罚。但是这些惩罚不应该是来自舆论，而是来自法律，从而继续完善法律，改变整个行业。

其实在各类招聘、信息分类网站上发布虚假信息已经是一个常态，甚至是行业的台面规则。虽然传统的招聘网站没有被爆出应聘人员被骗去传销，但是传统网站的大多数招聘信息都是骗应聘着到假的面试场所，然后骗取应聘着钱财。通过传统应聘网站上找到的应聘信息，最后被骗钱或者索要钱财的事情常有发生。

跟进一步说，我相信经过传统的应聘网站应聘，最后被骗到传销的事情发生的太多了，仅仅是因为在互联网不发达的群体里面，这样的时间被每天的时下热点掩埋罢了。

可以说互联网的本质就是通过解决信息不对称的问题来给用户提供服务，但是互联网信息不对称的问题并不会因为日益增多的互联网产品和服务而被解决，只是从原有的不对称变为新的信息不对称而已。

互联网背后真实信息，还是掌握在少数人的手上，一直都是如此。

Mac 终端切换目录

最近经常使用到 Mac 终端进行部署环境和安装包，需要给 Anaconda 里面安装jieba库的时候，发现 Anaconda 打开经常出错，所以只能使用终端进行安装。在终端里面需要打开硬盘下 anaconda/lib这个目录，然后执行 pip inatall jieba这个命令。打开终端，发现默认地址是我的 user 下的目录，使用ls进行查看的时候，发现并不是硬盘下面的目录。在网上找到一些关于 Mac 终端命令的文章，很多文章都没有提到怎么从user的跟目录切回到硬盘根目录。

找啊找，最后终于找到一篇文章提到这个切换的方式。

其实硬盘的根目录，就是user根目录的上一级目录，所以我们需要找到的是怎么切换回去当前目录的上一级目录，那篇博文中提到：

cd 切换工作目录
. 表示当前目录
.. 表示当前目录的上一级目录
/ 根目录/目录分隔符
./ 当前目录
../ 回到上一级目录
相对路径/绝对路径 凡是以/开始的路径，都是绝对路径
cd .. 进入上一级目录
cd 加波浪符号：进入当前登陆用户目录下

根据上面介绍，我们需要返回上一级目录，只需要输入cd ..即可。

虽然是一个很简单的问题，但是对于新手来说，这个问题就就如同前进道路的一座小山，只有翻过去才能继续前行。

圣费尔明节

发表于 2017-08-15 | 分类于 互联网 |

圣费尔明节（西班牙语：San Fermín）是属于西班牙纳瓦拉自治区首府潘普洛纳市的一项传统庆祝活动。该节日因为欧内斯特·海明威的著作《太阳照常升起》描写过其中的奔牛活动而闻名于世。

上面的内容是来自维基百科对圣费尔明节介绍，在第一财经周刊上看到关于圣费尔明节中奔牛活动的解读。奔牛活动是圣费尔明节最重要的庆祝活动，始于该节日活动的第二天早上，也就是 7 月 7 日早上。在当天早上 8 点，人们会穿着白色衣衫，戴着红色圣费尔明围巾和红腰带，手里拿着卷起的报纸，报纸是用于在牛接近的时候驱赶它们。

那么，为什么会有奔牛这样的活动呢？除了最初带有宗教的活动形式之外，据说当时政府为了刺激国内经济，或许维持这项活动。似乎社会上很多看似不太合理的活动，背后都存在一套理论支撑，或许是为了社会繁荣稳定，或者是为了政治管理。

在众多家长一致反对的游戏行业，现在逐渐得到社会的任何，之所以能发展起来，也是因为背后那套经济需要发展，人类需要娱乐的理论在支撑，如果政府当时一刀切，或许就没有现在的游戏行业，也就少了很多的经济支持。

虽然说的有点赤裸，但是这并不代表坏事，人类所有的进步都是来源于内心的不满足与贪婪，所有的停止不前都来源于目光短浅的满足。

目前国内的医药电商模式主要分为五种类型，分别是以天猫医药为代表医药电商平台、以 1 药网为代表的独立 B2C 商城、以七乐康为代表的以线上问诊带动医药销售的模式，还有以叮当、快方送药等 O2O，还有就是 B2B 模式。在业绩上看， To C 的医药电商成绩并不佳，对于整个药品终端市场说，还是显得微不足道。

很早之前，就听说医药电商要冲击实体药店，但是很多年过去的，0.4% 的占比怎么去冲击 实体的 25%。

专业主义的一些特征

发表于 2017-08-01 | 分类于 学习札记 |

日本学者大前研一有一本书，书的名字叫做《专业主义》，在书的开头有这么一个句话——专业主义：21世纪你唯一的生存之道。是的，在这个时代，专业不应该是我们的最终追求，而应该是我们对自己的基本要求。

在这本书中，大前研一介绍了什么是专业主义？专业主义的人都具有什么特质？一本书都是围绕这些内容来讲。今天刚好在吴军老师专栏——《硅谷来信》听到吴军老师对于专业的五点看法，所以整理一下，分享与您。

第一点，专业的人能控制自己的情绪。专业人士在任何情况下，都是将工作（或者说事情）本身考虑，不会因为自己的个人情绪或者外部环境而影响到自身的工作和对事情的态度。在专业主义这本书中也有提到，专业主义就是需要控制自己的情绪，并依靠理性而行动。

第二点，专业的人精通并且遵守行业流程和规范。在工作中，需要对自己面对的工作内容熟悉，并且精通该项工作的基本流程和规范，是专业素养的一个体现。一个专业的医生，在给病人看病的时候，需要根据临床路径和指南来给患者进行诊断和治疗。同样的，每个行业都需要有这样的规范，否则就会忽略某个环节而把事情搞砸。

第三点，专业的人充满好奇心（好学心）寻找更好的解决方案。专业的人，并不是简单解决问题，而是发现问题，找到解决方案，找到尽可能好的解决方案，当然这是在实现条件允许的情况下。不专业的人，会为了交差而敷衍了事。同样的，在《专业主义》中也提到，专业的人应该是充满好奇心，去寻找工作上更多的答案。

第四点，专业的人具备完整的知识体系。针对已知的问题，对于绝大多数人，只要有足够的时间和资源，都能把问题解决掉。但是对于未知问题，只能是那些具有该领域完整知识体系的人才能解决。这就是专业与不专业的区别，专业的人可以在现实预测未来可能发生的问题，不专业的人只能解决当下出现的问题。

第五点，专业的人能在动态中不断学习，进步。无论是从事什么行业，现在知识的变更越来越快，如果不学习，很快就会发现过去掌握的知识已经不再适用，也就称不上专业。这个观点和第三点是类似的，强调好奇心对专业主义的作用，只有不断进步，才能保持专业。

以上就是关于无论老师分享的关于专业主义的五点特质，不知道你满足了多少条？

SEO 中的爬行、抓取、索引和收录

发表于 2017-07-31 | 分类于 互联网 |

搜索引擎的基本原理：首先使用搜索引擎蜘蛛，沿着网站的 URL 「爬行」，每爬一个 URL，就保存该 URL 到待抓取库，然后访问待抓取库中的 URL，对该页面的内容进行「抓取」，解析 HTML 的内容，系统判断该页面内容的质量，考虑是否值得「收录」，如果值得收录，那就将该 URL 在搜索引擎系统中建立「索引」。

开头的这段话分别介绍四个动作，那就是「爬行」、「抓取」、「收录」和「索引」，也是今天这篇文章讨论的内容？

什么是爬行？
爬行是指搜索引擎的爬虫机器人，我们称之为「搜索引擎蜘蛛」，从已知网站的URL 出发，沿着网站的每个URL地址，新发现的 URL 进行不断的爬行，并将新发现的 URL 地址保存在待抓取的地址库中。

什么是抓取？
抓取是指访问在爬行中获取到的待抓取地址库中的 URL，并且将这些 URL 的页面进行解析，将其解析到的内容（HTML 代码）保存在数据库中。

什么是索引？
索引是指将前面解析出来，并且认为是「有价值内容」的页面，在数据库中建立对应关系，形成索引库。当用户在搜索引擎中针对某个关键词进行搜索，根据关键词和索引的关系，将页面反馈给到用户，这也是搜索引擎的基本工作原理。也就是说，只有添加了索引的 URL 才能被用户在搜索引擎中搜索出来。

什么是收录？
其实收录的意思就是指 URL 的信息在搜索引擎的索引库中建立了索引，所以收录和索引是一样的意思，索引是指机器索引，对于网页来说，是被机器收录。

nofollow 属性是什么？
在网站页面中，通常会存在一些链接，我们可以对这些链接设置nofollow属性。nofollow属性的意义就是告诉爬虫蜘蛛：这个 URL 地址不要继续爬行，就当作不是链接来处理。为什么会存在 nofollow属性呢？最根本的作用就是：对于外链，尤其是质量差外链（比如付费广告），站长需要添加 nofollow属性来减少网站 RP 值的外流，保持网站的权重。虽然爬虫蜘蛛在某个页面爬到某个 URL 有nofollow属性，停止爬行。但是在别的地方再遇到这个 URL 的时候，并且没有nofollow属性，这个 URL 还是会被蜘蛛爬行，抓取，甚至是建立索引，将其收录。

noindex 标签是什么？
在网页头部信息中，添加meta noindex标签信息，告诉搜索引擎，我这个网页不要添加索引，不要让用户在搜索引擎中搜索出来。具有noindex标签的页面，搜索引擎蜘蛛是会爬行并且抓取的，因为爬行和抓取是在索引的前面的，只是到解释到该页面的HTML内容的时候，发现存在noindex标签，只会将其内容保存起来，但是不会对页面建立索引，用户也就不能在搜索引擎搜索到该页面。

robots.txt 是什么？
robots.txt 亦称之为 robots 协议，一般存在于网站的根目录下，一般直接访问「www.xxxx.com/robots.txt」都能看到 robots.txt 的内容。robots 协议会告诉搜索引擎蜘蛛，该网站下哪些页面可以抓取，哪些页面不可以抓取。robots 协议并不是所有的搜索引擎都遵守，但是主流搜索引擎都是遵守 robots 协议的。所以说，robots 协议并不是行业规范，也是一种约定俗称。

最后几点补充
无论是nofollow 、noindex还是 robots 协议，当然还有其他的一些搜索引擎规则，这些属性、标签和协议，并不是一开始，所有的搜索引擎都遵守。比如说，对于 robots 协议，直到 2008 年，Google、雅虎、百度这些主流的搜索引擎才一致承认这样的协议，承诺自家的搜索引擎蜘蛛将会遵守 robots 协议上面的内容取爬取网站的内容。所以，搜索引擎就是一个不断完善，不断进步的产品，无论是技术还是准则，都是一个迭代的过程。
