2018 年阅读计划

发表于 2018-01-02 | 分类于 生活随笔 |

在昨天的文章中，提到今年需要精读 12 本书，并且额外增加 10 本作为扩展阅读。其实之前已经知道自己还需要读哪些书，所以早上就将这些书整理，放在豆瓣的豆列中。每个月都会对计划进行反馈，尤其是读书和写作计划，所以现在将12 本已经确定的书籍列出来，并对书籍进行简单介绍。

第 1 本，吴军老师的《文明之光》（3、4）册，其实这是 2017 就应该完成的阅读任务，但是当时只读了 1、2 两册，打算今年把后面两册读完。《文明之光》是一套关于世界文明发展的书籍，吴军老师从世界历史、科技、艺术和政治四个方面下手，用他丰富的知识体系和见识，写就这套人人称赞的图书。

第 2 本，《二手时间》，这是一本讲述苏联解体之后，俄罗斯普通人生活的纪实性文学，值得我们去细细品味。

第 3 本，《平面设计的网格系统》，一本设计书籍，主要介绍在平面设计中，网格的功能和使用方法，网格系统是为设计师们提供的实用性工具，从概念、组织结构和设计上更有效、自信地处理和解决视觉问题。因为今年的计划是学习产品书籍，所以入手这本书，作为平面设计构图的教材。

第 4 本，《智能时代》，也是吴军老师的作品。2017 年人工智能异常的火热，这是我们不得不关心的一个领域。市面上可能有很多关于人工智能的书籍，但是对于人工智能，并不是所有人都有说话权。吴军老师是计算机方面的科学家，并且也跟人工智能大师尼克教授学习过，可以说是人工智能方面的专家。

第 5 本，《斯蒂夫·乔布斯传》，是的，这本书我还没有来得及看。这个时代已经没有乔布斯，未来或许也不会再有乔布斯。

第 6 本，《穷查理宝典》，2018 年会面对着很大的财务压力，很大的原因是 2017 年的理财计划没有完成。所以希望能从这本书中，得到关于财务管理的智慧，理性决策自己的财务管理。

第 7 本，《支付战争》，可能这本书在 2015 年看更加合适，现在移动支付元年已经过去的，但是这并不代表过去的经验对未来不再适用。

第 8 本，《上帝的手术刀》，未来是生物技术的时代，作为一名药学专业人，我必须去了解更多关于生物制药、人类基因的问题。

第 9 本，《创作自然》，一本讲述科学家——洪堡的故事，对于这个人物，我了解甚少，但是还是抱着了解自然的心态去看这位科学家如何发现大自然。

第 10 本，《文明是副产品》，一本介绍文明为什么会产生的书籍，或许搭配《文明之光》一起阅读会更有乐趣。

第 11 本，《娱乐至死》和《理解未来的 7 个原则》，这是两本书，打算都安排在11 月份，在得到上听过《娱乐至死》的解读版，觉得里面的解读没有抓住作者强调的重点，所以想要自己读一遍。至于为什么会选择《理解未来的 7 个原则》，我想是因为自己对于未来的无知、恐惧和焦虑吧，所以尝试去理解未来。

第 12 本，《随意搜寻》这是一本设计产品设计和运营思考的书籍，作为自己工作知识的补充书籍，叔推荐的。

2018年计划与誓言

发表于 2018-01-01 | 分类于 生活随笔 |

刚写完 2017 年终总结，也来写写 2018 年的计划吧！吸取 2017 年计划完成不佳的教训，2018 年的计划需要更加细致一点，需要及时获取反馈，及时调整。

365 天，每天坚持写作和写日记
日记，用来记录每天发生的事情，可以是流水账的形式，也可以是随性记录的形式。每天需要拍摄一张图片，放在日记中，可以是醒来之后在阳台拍摄的天空，也可以是在路上随手拍摄的路人或者风景。每篇日记的字数不得低于400字，最好能写到 600 字。

写作，使用 Ulysses 记录下每天的灵感，将这些灵感进行延伸，每周写两篇公众号文章，推送到「第七周期」公众号上，作为内容输出。写作内容不做太多的限制，但不可盲目追求热点，写一些有思考性的东西。

争取在 6 月结束前将公众号订阅用户增长到 5000 人，这是检验写作质量的标准。

每周、每月去 Review 自己 OKR 的计划，并且将每月进展写成一篇文章，发表在「第七周期」博客上，不作为公众号内容推送。

365天，每天坚持阅读
2018 年的阅读计划是精读 12 本书，书名会在后面的文章给到。精读的书需要写成读书笔记，发布在「第七周期」公众号，每月一篇。需要在当月的 30 号天推送出去。额外增加 10 本，这是本书暂不确定，作为满足工作和其他学习任务的待定书籍。

每天阅读至少 15 页，按照这样的速度，一个月就能阅读 450 页，基本能完成每月两本书的任务。需要将每天读过内容浓缩成 500 字以内的摘要，也是作为写作训练的一部分。

每周坚持跑步两次，至少跑步 8 公里
运动不能每天都坚持，但是需要每周都坚持。每周两次跑步，每次最少 4 公里。在公司跑步机或者在江边跑，在跑步之余，可以增加部分力量训练，体重维持在 115 斤范围，不能低于 110 斤，更不能高于 120 斤。

完成三项任务
考虑到职业发展，需要在今年做 3 件事情，分别是产品设计、程序开发、注册会计师。

产品设计，主要是围绕产品交互设计，学习设计的基础知识，包括构图、排版、色彩搭配、设计心理学等知识，并且自己完成一款产品的设计稿，包括是 Web 端和无线端，无线端就是微信小程序。

程序开发，自己的主要工作还是产品。考虑到需要实现自己的 Demo，必须学习开发技术。主要围绕 Python 和小程序开发，把 2017 年没有完成的网站做起来，需要在 4 月底上线 Web 端。然后在 6 月前上线小程序版本，如果情况有变，提前上线小程序版本也是可以的。

程序开发和设计是一起交替进行，根据设计推进开发去解决问题，这个任务比较重，需要花费半年的时间（也就是6月底）才能出一个比较丑陋的版本。

注册会计师报名是在 4 月份，考试是在 10 月份。考虑到前面一项任务的时间，这项任务会在7月或者更晚一些进行，也有可能会取消。当然取消的前提是前面两项任务需要继续花费自己较大的时间，否则计划就需要继续进行下去。

本次注册会计需要考两门课程，分别是会计和审计，最晚会在 8 月份启动，并且考虑报班学习，争取一次通过这两门课程。

势如破竹
2018 年，需要做出一些改变，否则就会像 2017 年那样碌碌无为，除了以上这些计划之外，还需要坚持执行的有：

  1. 没有朋友在的情况下，需要晚上 12 点上床睡觉，早上 7 点起床；
  2. 每周坚持给爱人写信，写需要在每周日晚上寄出，直到他过来一起生活；
  3. 每天坚持记账，不需要其他的记账工具，使用支付宝即可，月底总结自己的每月消费情况，写成日记，争取提前完成自己的年度财务目标。
  4. 每周看一部电影，题材不限；
  5. 学习 2 门 MOOC 课程，并且需要获得优秀证书，两门课程会写在 OKR 中。

具体计划安排会在印象笔记中使用 OKR 进行记录，最重要的是每周每月 Review ，将每月的进展更新到博客中，这也是今年的计划和誓言。

照旧，每年的今天，来写写今年的总结吧！

年初，有制定 OKR 计划，但是执行效果并不佳，最后的计划都没有好好的执行，也没有按时去回顾，所以基本废了。

谈谈写作，原本计划每周的写作，最后也被耽搁了，原因有很多，年初是因为工作排的太慢，几乎天天加班，周末也加班，这么一放弃，年中换了工作环境，不再像过去那样加班了，但是长时间没写，要写起来就比较艰难，断掉的事情很难再启动去做。年底仔细分析吧，主要还是定力不足，诱惑太多，东一榔头，西一棒子的，整体完成度不够。

谈谈财务，今年原本制定了财务计划，按照原定的计划执行，效果很不错。但是在年终工作变动，对于财务有了压力，不得不改变计划。也就是这样一停，下半年就没有任何的财务规划了，也就见步行步。

7个月的时间下来，学到了很多，不能说这份工作总是能让我充满热情，但更多的时候还是有热情去做的，谈不上有太多的成就，但起码能给团队解决一些问题。

2018，预期而至！

互联网并没有改变信息不对称

发表于 2017-09-02 | 分类于 互联网 |

李文星之死，也揭开 Boss 直聘在履行平台审核上的漏洞，媒体和同行纷纷将舆论的矛头指向 Boss 直聘，对 Boss 直聘进行口诛笔伐。

Boss 直聘没有履行好平台的责任，确实存在责任，应该得到惩罚。但是这些惩罚不应该是来自舆论，而是来自法律，从而继续完善法律，改变整个行业。

其实在各类招聘、信息分类网站上发布虚假信息已经是一个常态，甚至是行业的台面规则。虽然传统的招聘网站没有被爆出应聘人员被骗去传销，但是传统网站的大多数招聘信息都是骗应聘着到假的面试场所，然后骗取应聘着钱财。通过传统应聘网站上找到的应聘信息，最后被骗钱或者索要钱财的事情常有发生。

跟进一步说，我相信经过传统的应聘网站应聘，最后被骗到传销的事情发生的太多了，仅仅是因为在互联网不发达的群体里面，这样的时间被每天的时下热点掩埋罢了。

可以说互联网的本质就是通过解决信息不对称的问题来给用户提供服务，但是互联网信息不对称的问题并不会因为日益增多的互联网产品和服务而被解决，只是从原有的不对称变为新的信息不对称而已。

互联网背后真实信息，还是掌握在少数人的手上，一直都是如此。

Mac 终端切换目录

发表于 2017-08-20 | 分类于 学习札记 |

最近经常使用到 Mac 终端进行部署环境和安装包，需要给 Anaconda 里面安装jieba库的时候，发现 Anaconda 打开经常出错，所以只能使用终端进行安装。在终端里面需要打开硬盘下 anaconda/lib这个目录，然后执行 pip inatall jieba这个命令。打开终端，发现默认地址是我的 user 下的目录，使用ls进行查看的时候，发现并不是硬盘下面的目录。在网上找到一些关于 Mac 终端命令的文章，很多文章都没有提到怎么从user的跟目录切回到硬盘根目录。

找啊找，最后终于找到一篇文章提到这个切换的方式。

其实硬盘的根目录，就是user根目录的上一级目录，所以我们需要找到的是怎么切换回去当前目录的上一级目录，那篇博文中提到：

cd 切换工作目录
. 表示当前目录
.. 表示当前目录的上一级目录
/ 根目录/目录分隔符
./ 当前目录
../ 回到上一级目录
相对路径/绝对路径 凡是以/开始的路径，都是绝对路径
cd .. 进入上一级目录
cd 加波浪符号：进入当前登陆用户目录下

根据上面介绍，我们需要返回上一级目录，只需要输入cd ..即可。

虽然是一个很简单的问题，但是对于新手来说，这个问题就就如同前进道路的一座小山，只有翻过去才能继续前行。

  5. 显示合并的方式，也就是 indicator 参数；

圣费尔明节

发表于 2017-08-15 | 分类于 互联网 |

圣费尔明节（西班牙语：San Fermín）是属于西班牙纳瓦拉自治区首府潘普洛纳市的一项传统庆祝活动。该节日因为欧内斯特·海明威的著作《太阳照常升起》描写过其中的奔牛活动而闻名于世。

上面的内容是来自维基百科对圣费尔明节介绍，在第一财经周刊上看到关于圣费尔明节中奔牛活动的解读。奔牛活动是圣费尔明节最重要的庆祝活动，始于该节日活动的第二天早上，也就是 7 月 7 日早上。在当天早上 8 点，人们会穿着白色衣衫，戴着红色圣费尔明围巾和红腰带，手里拿着卷起的报纸，报纸是用于在牛接近的时候驱赶它们。

每年的奔牛活动都会引来媒体的广泛关注，由于奔牛节的危险性，每年大约有200到300人，也就是3%的参与者会受到不同程度的伤害。自1924年至今，已有15人在奔牛活动中丧生。除此之外，动物保护组织强烈反对这种行为，每到活动期间，就会出现游行活动。

那么，为什么会有奔牛这样的活动呢？除了最初带有宗教的活动形式之外，据说当时政府为了刺激国内经济，或许维持这项活动。似乎社会上很多看似不太合理的活动，背后都存在一套理论支撑，或许是为了社会繁荣稳定，或者是为了政治管理。

在众多家长一致反对的游戏行业，现在逐渐得到社会的任何，之所以能发展起来，也是因为背后那套经济需要发展，人类需要娱乐的理论在支撑，如果政府当时一刀切，或许就没有现在的游戏行业，也就少了很多的经济支持。

虽然说的有点赤裸，但是这并不代表坏事，人类所有的进步都是来源于内心的不满足与贪婪，所有的停止不前都来源于目光短浅的满足。

最近一两年，越来越多的人使用 Hexo 搭建博客，有其是有了 Next 这个主题之后，更是一发不可收拾。我用了 3 年的 WordPress，也坚持写了一些东西。无疑 WordPress 社区氛围很好，很多问题到各种论坛、博客都能得到解决。但是 WordPress 有几个缺点，比如系统庞大，一般的虚拟主机运行慢。找不到让我满意的主题，尝试过国外的付费主题，还是不太满意。官方的主题有很多问题，比 MarkDown 渲染效果很差。其实最主要还是自己不会开发出来更好的主题。

当然，WordPress 有很多其他平台满足不了的。比如，它不仅可以搭建博客，还可以搭建企业网站，甚至是导购和购物网站，现在很多网站都是经过 WP 二次开发而来的，比如什么值得买，爱范儿等等。除此之外，很多 WP 搭建的占比对 SEO 比较友好，收录比较好。

所以，Hexo 来了，最早了解 Hexo 的时候，并不知道它的工作原理，直到今天，才知道这是一个没有后台的博客站点，其实原理通过 MarkDown 文件渲染，然后利用 GitHub Page 生成在外网的静态文件网站，每次更新文章内容的时候，需要通过 Git 提交到 GitHub 上。

了解的基本的信息，在网站找来一些教程，就开始折腾。好几个小时下来，总算是完成，过程中有太多的坑，躺下去，一段让我想要放弃，最后还是重拾信心，算是完成吧。

首先来谈一下环境搭建的问题，分别需要安装 Node.js，Git ，在这个过程中，不同的教程给出来的方法是不一样，所以有点乱，其实最好的方式还是直接到官网下载安装，下载完成之后就好。

另外一个坑就是配置 GitHub 上，由于空格的问题折腾好几个小时，最后还是解决。

把所有的问题解决好，发布到线上之后，准备迁移 WordPress 的文章，发现文章并不都是 MarkDown 语法生成的，只能一一修改，改了大概两个小时，总算是完成。

最后配上 Next 主题，配置主题语言、头像等，总算大功告成，改天再把域名加上，基本就完事。

至于 Hexo 这样的发布形式是不是符合写作习惯，有待考察。

医药电商是不是伪需求

发表于 2017-08-08 | 分类于 移动医疗 |


2005 年国内第一张医药电商牌照发放给药房网，后来到 2010 年牌照的数量激增，到 2016 年更是简化了 C 证的申请流程，似乎医药电商迎来春天，然而在战绩上，似乎并没有多少的亮点可言。

目前国内的医药电商模式主要分为五种类型，分别是以天猫医药为代表医药电商平台、以 1 药网为代表的独立 B2C 商城、以七乐康为代表的以线上问诊带动医药销售的模式，还有以叮当、快方送药等 O2O，还有就是 B2B 模式。在业绩上看， To C 的医药电商成绩并不佳，对于整个药品终端市场说，还是显得微不足道。

国内某个医药数据公司发布的报告显示，医药电商占整个终端销售总额 0.4%，传统零售药店占到 25% 左右，也就意味着，医院、社区医院、个人诊所等才是药品终端的主战场。

医药电商这么多年，所谓投入太多的社会资源和资金，如今占比也就不到 1%。在资源利用和资金变现上，确实不是一个好的高效率的东西。当我们了解到这些事实之后，证明互联网并不是什么万金油，有可能是绊脚石。

很早之前，就听说医药电商要冲击实体药店，但是很多年过去的，0.4% 的占比怎么去冲击 实体的 25%。
